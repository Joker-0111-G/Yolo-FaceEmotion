## 1. 项目背景与意义

### 1.1 什么是人脸表情识别？

人脸表情识别（Facial Expression Recognition, FER）是计算机视觉领域的重要研究方向。它通过分析人脸图像中的面部特征（眉眼、嘴角、鼻翼等肌肉运动模式），自动判断人的情绪状态。

基于 **Ekman 基本情绪理论**，系统识别 **7 种基本表情**：

| 😠 Anger | 🤢 Disgust | 😨 Fear | 😊 Happy | 😐 Neutral | 😢 Sad | 😲 Surprise |
|:--------:|:----------:|:-------:|:--------:|:----------:|:------:|:-----------:|
| 愤怒 | 厌恶 | 恐惧 | 开心 | 中性 | 伤心 | 惊讶 |

### 1.2 应用场景

| 领域 | 应用 | 价值 |
|:-----|:-----|:-----|
| 🎓 **智慧教育** | 课堂学生专注度与情绪分析 | 实时掌握教学效果，调整授课策略 |
| 🏥 **心理健康** | 抑郁症等情绪障碍辅助筛查 | 低成本、无感式的早期预警 |
| 🚗 **智能驾驶** | 驾驶员疲劳/情绪状态监测 | 降低因情绪波动导致的交通事故 |
| 🤖 **人机交互** | 情感计算与智能客服系统 | 更具同理心的人机对话体验 |
| 🛒 **新零售** | 顾客满意度无感采集 | 无需问卷即可获取消费情绪反馈 |

## 2. 项目简介

**Yolo-FaceEmotion** 是一个端到端的人脸表情识别系统，基于 YOLOv8s-cls 分类模型和 FER2013 数据集构建，实现了从**数据集处理 → 模型训练 → 实时推理**的完整流程。

### 核心功能模块

## 4. 技术方案与模型架构

### 4.1 YOLOv8s-cls 分类模型

- YOLOv8 是 Ultralytics 公司开发的最新计算机视觉框架
- 本项目使用 **YOLOv8s-cls**：
  - `s` = small（轻量版，兼顾精度与速度）
  - `cls` = 分类任务专用变体

### 4.2 迁移学习策略

本项目是**迁移学习（Transfer Learning）**在计算机视觉中的典型实践：

```
┌──────────────────────────────────────────────────┐
│                                                   │
│  ImageNet (百万级图片, 1000类)                     │
│       │                                           │
│       ▼                                           │
│  yolov8s-cls.pt ← 预训练权重                       │
│       │                                           │
│       │  1. 加载预训练权重                          │
│       │  2. 冻结 156/158 层骨干网络                 │
│       │  3. 替换分类头：1000类 → 7类                │
│       ▼                                           │
│  FER2013 微调 (200 epochs, 2.1小时)                │
│       │                                           │
│       ▼                                           │
│  best.pt ← 最优模型 (71.61% Top-1)                 │
│                                                   │
└──────────────────────────────────────────────────┘
```

### 4.3 技术选型理由

| 候选方案 | 对比分析 | 结论 |
|:---------|:---------|:-----|
| 传统 CNN (LeNet/AlexNet/VGG) | 架构陈旧，特征提取能力弱 | ❌ |
| ViT (Vision Transformer) | 需海量数据预训练，FER2013 上难收敛 | ❌ |
| ResNet / EfficientNet | 纯分类框架，需手动实现数据增强等 | △ |
| **YOLOv8s-cls** | 内置 RandAugment，完整工具链，生态成熟 | ✅ |

‌RandAugment 是谷歌 2019 年推出的自动化数据增强技术，仅用 2 个参数就能显著提升深度学习模型精度

**选择 YOLOv8 的核心理由：**

- ✅ 内置 **RandAugment** 数据增强策略，一个参数即可启用
- ✅ **Ultralytics 框架**提供完整的 `train/val/predict/export` 工具链
- ✅ 推理速度 **0.3ms/张**，满足实时视频流处理需求
- ✅ 社区活跃，文档齐全，学习曲线友好

## 8. 实时推理系统

### 8.1 系统代码

```python
import cv2
from ultralytics import YOLO

def run_realtime_system():
    # 1. 加载情绪分类引擎
    emotion_model = YOLO('runs/classify/Emotion_Sys/v1_model/weights/best.pt')

    # 2. 加载 OpenCV 内置的人脸检测器
    face_cascade = cv2.CascadeClassifier(
        cv2.data.haarcascades + 'haarcascade_frontalface_default.xml'
    )

    # 3. 启动本地摄像头
    cap = cv2.VideoCapture(0)

    if not cap.isOpened():
        print("警告：无法打开摄像头！")
        return

    print("系统启动成功！对准摄像头做表情吧。按键盘 'q' 键退出。")

    while True:
        ret, frame = cap.read()
        if not ret:
            break

        # 转为灰度图，大幅提升人脸检测速度
        gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)

        # 检测画面中的人脸
        faces = face_cascade.detectMultiScale(
            gray, scaleFactor=1.1, minNeighbors=5, minSize=(60, 60)
        )

        for (x, y, w, h) in faces:
            # 向外扩展截取框，防止关键特征被切掉
            padding = 15
            y1 = max(0, y - padding)
            y2 = min(frame.shape[0], y + h + padding)
            x1 = max(0, x - padding)
            x2 = min(frame.shape[1], x + w + padding)

            # 抠出人脸区域
            face_crop = frame[y1:y2, x1:x2]
            if face_crop.size == 0:
                continue

            # YOLO 前向推理
            results = emotion_model(face_crop, verbose=False)

            # 提取最高置信度结果
            top_idx = results[0].probs.top1
            emotion = results[0].names[top_idx]
            confidence = results[0].probs.top1conf.item()

            # 绘制绿色识别框和文字
            cv2.rectangle(frame, (x1, y1), (x2, y2), (0, 255, 0), 2)
            label = f"{emotion} ({confidence * 100:.1f}%)"
            cv2.putText(frame, label, (x1, y1 - 10),
                       cv2.FONT_HERSHEY_SIMPLEX, 0.7, (0, 255, 0), 2)

        cv2.imshow('YOLO Emotion Recognition', frame)

        if cv2.waitKey(1) & 0xFF == ord('q'):
            break

    cap.release()
    cv2.destroyAllWindows()
```

### 8.4 工程实现亮点

- ✅ **灰度转换加速**：人脸检测在灰度图进行，减少 2/3 计算量
- ✅ **`verbose=False`**：抑制每帧控制台推理输出，避免刷屏影响性能
- ✅ **`padding=15px`**：向外扩展人脸框，防止额头、下巴等关键区域被截断
- ✅ **边界保护**：使用 `max(0, y-padding)` 等方式确保裁剪不越界

## 11. 技术挑战与解决方案

### 挑战一：类别严重不平衡

| 问题 | 方案 |
|:-----|:-----|
| Disgust 类仅 436 张训练图，Happy 类 7,215 张<br/>最大/最小类比例 = **16.5 : 1** | ✅ 使用 RandAugment 对所有类别统一增强<br/>🔜 未来：对少数类过采样（SMOTE）或 Focal Loss |

### 挑战二：低分辨率图片

| 问题                                                         | 方案                               |
| :----------------------------------------------------------- | :--------------------------------- |
| FER2013 原图仅 48×48 灰度图<br/>面部细节有限，微妙差异难以捕捉 | ✅ 上采样至 64×64，保留更多面部特征 |

### 挑战三：Haar Cascade 人脸检测局限

| 问题 | 方案 |
|:-----|:-----|
| 对侧脸、遮挡、极端光照检测率低<br/>人脸检测失败 → 整个系统无法工作 | ✅ 当前使用 Haar（速度优先）<br/>🔜 未来：替换为 YOLO-face 检测模型 |

## 12. 未来优化方向

### 🔬 数据层面

- 对少数类（Disgust）过采样至与 Happy 相当的量级
- 引入更多真实场景数据集（RAF-DB, AffectNet）
- 尝试使用 GAN 生成少数类表情图片
- 使用更强的数据增强策略：CutMix + MixUp

### 🏗️ 模型层面

- 尝试 YOLOv8m-cls / YOLOv8l-cls 等更大模型
- 对比 ViT / EfficientNet 等架构在该任务的表现
- 引入注意力机制增强关键面部区域权重
- 探索多任务学习：同时预测表情 + 面部关键点

### 🔍 人脸检测升级

- 支持多角度人脸检测（侧脸、低头、仰头）
- 添加人脸对齐预处理
- 引入人脸质量评估过滤低质量输入
